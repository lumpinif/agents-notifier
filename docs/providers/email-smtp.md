# Email SMTP

中文文档：[email-smtp.zh-CN.md](email-smtp.zh-CN.md)

Use Email SMTP when you want Agents Router updates sent as plain text email through your own SMTP server or email provider.

Agents Router submits email to an SMTP server. It does not receive email, read inboxes, manage OAuth, or send attachments.

## Official Links

- [SMTP Message Submission, RFC 6409](https://www.rfc-editor.org/rfc/rfc6409)
- [Cleartext Considered Obsolete, RFC 8314](https://www.rfc-editor.org/rfc/rfc8314)
- [Internet Message Format, RFC 5322](https://www.rfc-editor.org/rfc/rfc5322)
- [Google Workspace SMTP settings](https://support.google.com/a/answer/176600?hl=en)
- [Microsoft 365 SMTP client submission](https://learn.microsoft.com/en-us/troubleshoot/exchange/email-delivery/fix-issues-with-printers-scanners-and-lob-applications-that-send-email-using-off)
- [Amazon SES SMTP interface](https://docs.aws.amazon.com/ses/latest/dg/smtp-connect.html)

## What You Need

- An SMTP host.
- A port.
- A TLS mode.
- A sender email address.
- One or more recipient email addresses.
- SMTP username and password, unless your SMTP relay explicitly allows no authentication.
- Agents Router installed.

## 1. Choose TLS Mode

Use one of these:

```text
starttls
implicit_tls
```

Recommended default:

```text
host = "smtp.example.com"
port = 587
security = "starttls"
```

Use `implicit_tls` only when your provider explicitly gives you port 465.

Agents Router V1 does not support remote cleartext SMTP.

## 2. Prepare Credentials

For many providers, the SMTP password is not your normal account password.

Common examples:

- Gmail or Google Workspace: app password or Workspace SMTP relay credentials.
- Microsoft 365: SMTP AUTH must be enabled for the mailbox or tenant.
- Amazon SES: SMTP credentials generated by SES.
- SendGrid: username is often `apikey`, and password is the API key.
- Mailgun: SMTP credentials from the sending domain settings.

## 3. Connect Agents Router

Run:

```bash
agents-router setup
```

Choose:

```text
Email SMTP
```

Enter the SMTP host, security mode, port, username/password if needed, sender, recipients, and optional Reply-To.

Agents Router stores the provider, starts the local service, and sends a test email through the same service route used by real agent events.

## Answer Detail

Email SMTP does not have the same small message limit as phone/chat providers.

Agents Router allows `Preview` or `Full Answer` for Email SMTP.

## Prompt Detail

Agents Router allows prompt detail for Email SMTP, but keep privacy in mind. Email may pass through your provider's mail servers and recipient inbox storage.

Prompt detail stays off by default.

## Manual Config

Email SMTP is configured in:

```text
~/.config/agents-router/config.toml
```

Simple authenticated config:

```toml
[[providers]]
id = "email_smtp"
type = "email_smtp"
host = "smtp.example.com"
port = 587
security = "starttls"
username = "alerts@example.com"
password = "<your SMTP password or API key>"
from = "Agents Router <alerts@example.com>"
to = ["you@example.com"]

[[routes]]
sources = ["codex_desktop"]
providers = ["email_smtp"]

[[routes]]
sources = ["agents_router"]
providers = ["email_smtp"]
```

Advanced: `username_env` and `password_env` are supported, but only use them when the environment variables are visible to the running local service.

```toml
[[providers]]
id = "email_smtp"
type = "email_smtp"
host = "smtp.example.com"
port = 587
security = "starttls"
username_env = "AGENTS_ROUTER_EMAIL_SMTP_USERNAME"
password_env = "AGENTS_ROUTER_EMAIL_SMTP_PASSWORD"
from = "Agents Router <alerts@example.com>"
to = ["you@example.com", "team@example.com"]
reply_to = "reply@example.com"
```

No-auth relay config:

```toml
[[providers]]
id = "email_smtp"
type = "email_smtp"
host = "smtp-relay.internal.example.com"
port = 587
security = "starttls"
from = "Agents Router <alerts@example.com>"
to = ["you@example.com"]
```

The running service automatically reloads valid config changes. If it is not running, start it:

```bash
agents-router start
```

## Delivery Semantics

SMTP success means the SMTP server accepted the message for delivery. It does not guarantee the message reached the final inbox.

Agents Router maps SMTP status classes like this:

- 2xx: sent.
- 4xx: temporary provider failure, retriable.
- 5xx: permanent provider rejection, not immediately retried.

## If It Does Not Show Up

Check these first:

- The SMTP host is a hostname, not a URL.
- The port matches the TLS mode.
- The username/password or SMTP API key is exact.
- The sender address is allowed by your SMTP provider.
- The recipient is allowed by your SMTP provider or relay.
- SMTP AUTH is enabled if you use Microsoft 365 authenticated submission.
- If you use `username_env` or `password_env`, the environment variable is visible to the running service.
- The local service is running:

```bash
agents-router status
```
